# week-1-retail-sales-excel-analysis

## Project Objective
This project is part of the LogicStack Data Analysis Internship (June 2026).
The goal is to learn how to clean, format, and analyze a real retail sales 
dataset using Microsoft Excel, and to practice basic formulas, sorting, 
filtering, and chart creation.

## Tools Used
- Microsoft Excel / Google Sheets
- GitHub (for submission)

## Dataset Columns
The dataset contains 1,000 retail transactions with these columns:
- Transaction ID
- Date
- Customer ID
- Gender
- Age
- Product Category
- Quantity
- Price per Unit
- Total Amount

## Work Completed
- Explored and understood the dataset structure
- Cleaned and formatted the data (table, bold headers, currency/date format, frozen header row)
- Checked data quality (blank cells, duplicates, valid values)
- Added Calculated Total and Amount Check columns using formulas
- Calculated key metrics using SUM, AVERAGE, MIN, MAX, and COUNT
- Created summary tables using SUMIF (sales by category, gender, quantity by category)
- Applied sorting and filtering, and captured screenshots
- Built Bar, Column, and Pie charts to visualize the data
- Wrote observations based on the analysis

## Key Observations
1. Electronics had the highest total sales (Rs. 156,905), slightly ahead of Clothing (Rs. 155,580), while Beauty had the lowest (Rs. 143,515).
2. Beauty had the lowest sales even though its quantity sold was close to the other categories.
3. Female customers generated more total sales (Rs. 232,840) than male customers (Rs. 223,160).
4. Customer ages range from 18 to 64, showing a wide adult customer base.
5. Out of 1,000 transactions, total quantity sold was 2,514 units, averaging 2-3 items per transaction.
6. The highest single transaction was Rs. 2,000, much higher than the average sale of Rs. 456.
